Common Windows Error Codes

Quick lookup of frequent Windows error codes with explanations and proven fixes.

Error Code / Message Explanation Basic Fix / Action
0x80070002 / 0x80070003 (Update) Update cannot complete; files missing/corrupted. Run Windows Update Troubleshooter. Delete files in C:\Windows\SoftwareDistribution\Download, retry.
0x800F081F (DISM/SFC/Update) Required system files missing or damaged. Run DISM/SFC (see System Maintenance). Repair install if needed with Media Creation Tool.
0xc000000f (Boot Error) Boot Configuration Data missing or corrupt. Boot from USB/DVD ‚Üí Repair your computer ‚Üí Command Prompt:
bootrec /rebuildbcd
bootrec /fixmbr
bootrec /fixboot
INACCESSIBLE_BOOT_DEVICE Windows can't access system drive. Check SATA/RAID BIOS settings. Use bootrec commands. Install correct storage drivers if new hardware.
CRITICAL_PROCESS_DIED Essential Windows process failed. Boot to Safe Mode, undo recent changes, run SFC/DISM. Scan for malware/hardware issues.
IRQL_NOT_LESS_OR_EQUAL Driver/RAM related issue. Update/downgrade drivers. Run RAM test (mdsched.exe). Remove recent hardware.
Blue Screen: KERNEL_SECURITY_CHECK_FAILURE Driver, hardware, or file corruption. Check stop code. Update drivers. Run sfc /scannow, chkdsk, dism /online /cleanup-image /restorehealth.

Tip: Always note the exact stopcode for reliable troubleshooting.

Safe Mode & Advanced Startup

Classic Method (F8)

On older systems, press F8 immediately after turning on for Safe Mode options.
Note: On modern SSD PCs, boot is often too fast for F8.

Modern Windows Method (10/11)

  1. Hold Shift and click Restart in the Start menu.
  2. Select Troubleshoot → Advanced Options → Startup Settings.
  3. Click Restart.
  4. Press 4 or F4 for Safe Mode, 5 or F5 for Safe Mode with Networking.

Useful Troubleshooting Commands

bcdedit /enum all
sfc /scannow
chkdsk C: /f /r
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd

Tip: Use Safe Mode for malware scans or to remove problematic drivers/software.

Battery Diagnostics

Windows Built-in Battery Report

Generate a detailed battery report via command:

powercfg /batteryreport /output %USERPROFILE%\battery-report.html /duration 7

Open battery-report.html in your browser. Check ‚ÄúDesign Capacity‚Äù vs ‚ÄúFull Charge Capacity‚Äù, cycle count, and history.

BatteryViewInfo (NirSoft)

Portable tool, no install needed.

Download here: nirsoft.net/utils/battery_information_view.html

  • Extract and run BatteryView.exe.
  • See battery ID, design/current capacity, voltage, charge %, cycle count, status. Export for support/warranty.

Battery Optimization Tips

  • Close background apps, dim screen while on battery.
  • Enable battery saver mode.
  • Try to keep charge between 20%-80%. Avoid full discharge or always plugged in.
  • Calibrate battery occasionally (full drain, then full charge).

Network Diagnostics & DNS

DNS Explanation

The Domain Name System (DNS) translates website names (e.g., www.example.com) to IP addresses that computers use.

What is an APIPA Address?

APIPA (Automatic Private IP Addressing) is an IP assigned by Windows when it cannot get an IP from a DHCP server.
It uses the IP range 169.254.0.1 to 169.254.255.254, e.g., 169.254.254.255.
If your network adapter has such an IP, it usually means no DHCP connection and a network problem.

Using ipconfig to Check Your IP Address

Open Command Prompt (Windows + R ‚Üí type cmd ‚Üí Enter) and run:

ipconfig

Look for the IPv4 Address under your network adapter. If it begins with 169.254, you have an APIPA address, meaning no DHCP IP assignment.

Troubleshoot by checking connections, DHCP server availability, or static IP settings.

Using nslookup

nslookup helps verify which DNS servers you are using and resolve domain names.

nslookup www.google.com

The result shows the IP address and the DNS server used for the lookup.

Set Static IP and DNS via ncpa.cpl

  1. Press Windows Key + R, type ncpa.cpl, then press Enter.
  2. Right-click your active network connection and choose Properties.
  3. Select Internet Protocol Version 4 (TCP/IPv4) and click Properties.
  4. Select Use the following IP address, enter IP address, subnet mask, and default gateway.
  5. Select Use the following DNS server addresses and enter your preferred DNS servers (e.g., Google DNS: 8.8.8.8 and 8.8.4.4).
  6. Confirm by clicking OK and close all dialogs.

Tip: Ensure these settings fit your network configuration and don't conflict with other devices.

  • Run Windows audio troubleshooter.
  • Update or reinstall sound drivers via Device Manager.
  • Re-plug or test other speakers/headphones.

Performance Bottleneck Analysis

Use Task Manager (Ctrl+Shift+Esc), Performance tab, to diagnose slowness:

CPU:

  • Very low speed (e.g. 0.10 GHz): power saving/throttle issue. Plug in charger. Check cooling!

Disk:

  • 100% usage, high response times: failing/old HDD. Upgrade to SSD recommended.

Memory:

  • High and steady RAM use: many apps open or memory leak. Close unused apps or add RAM.

Latency:

  • High disk latency may indicate hardware or driver problems.

Summary

  • Low CPU speed + slowness = power/thermal issues.
  • High disk usage + sluggishness = failing disk or RAM shortage.
  • High latency = drive or hardware issue.
  • GPU near 100% = driver bugs or hidden background apps.

Boot Repair & Partition Commands

Diskpart Commands (inspect disks & partitions):

diskpart
list disk
select disk 0
list partition
select partition 1
detail partition
exit

Boot Repair Commands:

  1. Boot from Windows Recovery USB/DVD.
  2. Choose Repair your computer → Troubleshoot → Command Prompt.
  3. Enter commands below by clicking copy:
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd

If EFI / System Partition Is Missing/Corrupted:

After assigning a drive letter (e.g., S:) to the EFI partition, run:

bcdboot C:\Windows /s S: /f UEFI

Windows Event Viewer

Event Viewer helps find hidden or recurring errors. Use these steps:

  1. Open Event Viewer via Start search.
  2. Go to Windows Logs → System.
  3. Click Filter Current Log.
  4. Check Critical and optionally Error under Event Level.
  5. Look for repeated critical errors and review details carefully.

Note: Most warnings are usually safe to ignore for troubleshooting.
